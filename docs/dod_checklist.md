# Definition of Done (DoD) - Plantilla

Cada historia debe cumplir los siguientes puntos antes de marcarse como `Done`:

- [ ] Código revisado y aprobado (Pull Request revisado)
- [ ] Tests unitarios relevantes añadidos y que pasan
- [ ] Tests de integración (si aplica) añadidos y que pasan
- [ ] Documentación actualizada (Postman / README / CHANGELOG)
- [ ] Demo funcional de la funcionalidad en entorno local
- [ ] No errores críticos en logs y linter limpio
- [ ] Issue actualizada con notas de despliegue o configuración especial

Usa esta checklist para cada issue y copia como subtareas si lo deseas.
